面向对象三大特征：封装、继承、多态。

extends继承或者implements实现，是多态性的前提。

1. 多态：一个对象拥有多种形态，就是对象的多态性

2. 多态的格式和使用

   代码当中体现多态性，其实就是一句话：父类引用指向子类对象。

   格式：（左侧父类的引用，指向了右侧子类的对象）

   `父类名称 对象名 = new 子类名称();`

   或者

   `接口名称 对象名 = new 实现类名称();`

3. 多态中成员变量的使用特点

   访问成员变量的两种方式：

   - 直接通过**对象名称访问成员变量：看等号左边是谁，优先用谁**，没有像上找
   - 间接通过**成员方法访问成员变量：该方法属于谁，优先用谁，**没有向上找

4. 多态中成员方法的使用特点

   访问规则：看 new 的是谁，就优先用谁，没有则向上找

   【口诀】：

   **成员变量：编译看左边，运行还看左边**

   **成员方法：编译看左边，运行看右边**
